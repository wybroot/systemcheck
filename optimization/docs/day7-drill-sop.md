# Day 7 容灾演练 SOP（备份恢复 + 故障切换）

目标：验证“可恢复”而不是只验证“有备份”。

## 1. 验收目标（DoD）
- 完成一次备份恢复演练并记录 RTO/RPO。
- 完成一次故障切换演练并记录业务影响窗口。
- 形成 Runbook 最终版。

## 2. 演练范围
- 备份恢复：数据库/配置/关键文件
- 故障场景：单实例故障、依赖不可用、网络隔离（择一或多项）
- 验证项：探活、核心接口、数据一致性

## 3. 演练步骤
1) 明确演练窗口与通知
2) 设定观测面板与记录员
3) 执行故障注入或恢复动作
4) 记录起止时间与关键事件
5) 演练后复盘并修订 Runbook

## 4. 关键记录字段
- 开始时间、恢复时间、恢复完成时间
- RTO（目标与实测）
- RPO（目标与实测）
- 失败点与改进项

## 5. 交付物
- 《Day7 演练记录》
- 《RTO/RPO 达成评估》
- 《Runbook v1.0》
