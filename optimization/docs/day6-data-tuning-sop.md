# Day 6 数据层调优 SOP（MySQL/Redis）

目标：降低慢查询与资源抖动，提升命中率与稳定性。

## 1. 验收目标（DoD）
- MySQL 慢查询数量下降，关键 SQL 有治理结论。
- Redis 命中率与内存策略明确，bigkey/hotkey 有跟踪。
- 备份链路可用，恢复路径可执行。

## 2. MySQL 重点
- 开启慢查询日志，持续采样。
- 对高频慢 SQL 做索引与执行计划优化。
- 控制连接数上限，避免过载。
- 关注锁等待、死锁、复制延迟。

## 3. Redis 重点
- 设置 `maxmemory` 与淘汰策略。
- 识别 bigkey/hotkey，避免单点热点。
- 检查慢命令、内存碎片率。
- 明确 AOF/RDB 持久化与恢复流程。

## 4. 执行流程
1) 采集现状
2) 确定高价值优化项
3) 灰度执行
4) 观察对比
5) 固化配置
