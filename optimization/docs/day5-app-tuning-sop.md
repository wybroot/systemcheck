# Day 5 应用层调优 SOP

目标：在不改业务语义的前提下，优化应用延迟、错误率与稳定性。

## 1. 验收目标（DoD）
- 核心服务 P95/P99 有可量化改善，错误率不升高。
- 线程池/连接池/超时参数有明确上限。
- 已配置重试退避、熔断或限流策略。

## 2. 执行策略
1) 定位瓶颈（CPU、锁、IO、下游依赖）
2) 优先调“池”和“超时”，后调运行时参数
3) 单服务灰度，观察后推广

## 3. Java 基线建议
- `Xms=Xmx`
- 使用 G1，保留 GC 日志
- 线程池有界队列 + 拒绝策略
- 下游调用统一超时

## 4. Python 基线建议
- Gunicorn worker 初值 `2*CPU+1`（同步模型）
- 明确 `timeout/graceful_timeout`
- 避免阻塞调用堆积

## 5. 通用保护机制
- 超时：连接、读、写分离配置
- 重试：仅幂等请求 + 指数退避
- 熔断：防止下游故障扩散
- 限流：保护核心资源
